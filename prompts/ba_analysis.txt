Analyse the following Jira ticket and produce a structured, developer-ready implementation specification.

JIRA TICKET:
ID: {ticket_id}
Summary: {ticket_summary}
Labels: {ticket_labels}

Description:
{ticket_description}

Existing Comments:
{ticket_comments}

AVAILABLE REPOSITORIES (key: repo — keywords that indicate changes belong here):
{available_repos}

ACTUAL REPOSITORY STRUCTURES (use these to generate correct file paths — do not guess):
{repo_structures}

INSTRUCTIONS:
1. Identify the core functional requirement from all available fields — summary, description, comments, and labels.
2. Break the requirement into discrete, testable acceptance criteria (Given/When/Then format). Each criterion must be independently verifiable.
3. Determine which repos need changes by matching the ticket content against the repo keywords above. A full-stack ticket will target multiple repos (e.g. ["api", "ui"]).
4. List every file that needs to change. For each file, set "repo" to the repo key it belongs to. Use the ACTUAL REPOSITORY STRUCTURES above to derive real paths — match the language, directory layout, and naming conventions already present in the repo. Never guess a language or path structure.
5. Set "target_repos" to the list of repo keys that need changes (derived from step 3).
6. Specify exact function signatures with type hints.
7. Identify edge cases the developer must handle explicitly.
8. If ANY information is ambiguous or missing, set "needs_clarification" to true and list your questions.
9. Estimate complexity: "low" (< 2h), "medium" (2–8h), "high" (> 8h).

OUTPUT FORMAT:
Respond with a single JSON object only. No text outside the JSON.

```json
{{
  "ticket_id": "{ticket_id}",
  "needs_clarification": false,
  "clarification_questions": [],
  "summary": "One-sentence description of what needs to be built",
  "acceptance_criteria": [
    "Given X, when Y, then Z",
    "Given A, when B, then C"
  ],
  "target_repos": ["api"],
  "files_to_change": [
    {{"path": "calculator.py", "repo": "api", "reason": "Add divide() function"}},
    {{"path": "tests/test_calculator.py", "repo": "api", "reason": "Add tests for divide()"}}
  ],
  "function_signatures": [
    "def divide(a: float, b: float) -> float"
  ],
  "edge_cases": [
    "Division by zero must raise ValueError with message 'Cannot divide by zero'"
  ],
  "complexity": "low",
  "implementation_notes": "Any additional context for the developer"
}}
```

{{
